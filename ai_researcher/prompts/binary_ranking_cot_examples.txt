paper 1:
Adaptive Tile-Based Shading for Mobile GPUs:
  Title: Adaptive Tile-Based Variable Rate Shading for Energy-Efficient Mobile Rendering
  Problem Statement: Mobile GPUs use tile-based rendering architectures but apply uniform shading rates across all tiles, wasting compute on low-complexity regions.
  Motivation: By analyzing per-tile content complexity using depth discontinuities, normal variation, and motion vectors at minimal cost, we can dynamically assign shading rates that maximize visual quality per watt.
  Proposed Method: A lightweight tile complexity analysis compute pass scores each tile and assigns 4x4 coarse, 2x2 coarse, or full-rate shading. Temporal smoothing prevents flickering.
  Step-by-Step Experiment Plan: Test on Sponza and Bistro scenes on Snapdragon 8 Gen 2, Mali-G715, and Apple A17 devices. Measure FPS, power, PSNR/SSIM.

paper 2:
Simple Resolution Scaling for Mobile:
  Title: Dynamic Resolution Scaling with Fixed Thresholds for Mobile Games
  Problem Statement: Mobile games need to maintain frame rate targets while preserving visual quality.
  Motivation: Current DRS approaches use simple frame time feedback to adjust resolution, but fixed thresholds lead to oscillation.
  Proposed Method: Monitor frame time over a sliding window and scale resolution up/down when the average exceeds thresholds. Use hysteresis to prevent oscillation.
  Step-by-Step Experiment Plan: Test resolution scaling on 3 mobile games. Measure average FPS and user survey results.

Meta-review:
Paper 1 proposes a technically novel approach to variable rate shading specifically designed for mobile tile-based architectures. It leverages multiple complexity signals (depth, normals, motion) and includes temporal stability mechanisms. The experiment plan is comprehensive with specific devices, scenes, and both quality and performance metrics. This represents a meaningful contribution to mobile rendering optimization.

Paper 2 proposes a straightforward extension of dynamic resolution scaling with hysteresis. While practical, this is essentially an engineering improvement of existing DRS systems and lacks novelty. The experiment plan is underspecified with only "user survey results" for quality evaluation. This would be below the acceptance threshold for a top graphics venue.

The accepted paper is paper 1.

paper 1:
Temporal Super-Resolution for Mobile Games:
  Title: MobileSR: Lightweight Temporal Super-Resolution for Tile-Based Mobile GPUs
  Problem Statement: Mobile games render at reduced resolution, producing blurry output. Desktop temporal upsampling is too expensive for mobile.
  Motivation: Design temporal upsampling specifically for tile-based mobile architectures using jittered rendering with lightweight recurrent filtering.
  Proposed Method: Render at lower resolution with sub-pixel jitter, accumulate into history buffer using motion vector reprojection with confidence-weighted blending and edge-directed spatial fallback.
  Step-by-Step Experiment Plan: Test 2x and 4x upsampling on mobile devices. Compare against bilinear, FSR 1.0, TAA. Measure PSNR/SSIM, frame time, power.

paper 2:
Neural Texture Compression for Mobile:
  Title: MobileNTC: Neural Texture Compression Optimized for Mobile GPU Architectures
  Problem Statement: Mobile devices have limited GPU memory and bandwidth, making texture storage a bottleneck.
  Motivation: Design neural texture representation optimized for mobile GPU tile-based memory hierarchy.
  Proposed Method: Encode textures as compact latent grid + tiny decoder network (<50K params). Decoder uses 1x1 convolutions efficient on mobile GPUs. Hardware-aware FP16/INT8 quantization.
  Step-by-Step Experiment Plan: Compare against ASTC at various quality levels. Measure compression ratio, decompression speed, PSNR/SSIM/LPIPS, GPU memory savings.

Meta-review:
Paper 1 addresses a high-impact problem in mobile gaming with a well-designed temporal approach. The confidence-weighted blending with disocclusion detection and edge-directed fallback shows careful engineering. The evaluation is thorough with multiple upsampling ratios and comprehensive metrics. This could significantly improve mobile gaming visual quality.

Paper 2 presents an interesting application of neural compression to mobile textures. The approach of using tiny decoders with hardware-aware quantization is practical. However, the per-texture training requirement and the overhead of running a neural network per texture fetch may limit practical adoption. The experiment plan is solid but the practical deployment challenges are underexplored.

Both papers have merit, but Paper 1 addresses a more immediately impactful problem with a more complete solution.

The accepted paper is paper 1.

paper 1:
Lightweight Probe-Based Global Illumination for Mobile:
  Title: Sparse Temporal Irradiance Probes for Real-Time Global Illumination on Mobile
  Problem Statement: Real-time GI on mobile is infeasible with current methods, making scenes look flat.
  Motivation: Sparse probes with temporal amortization and simplified updates can provide convincing GI within mobile budgets.
  Proposed Method: Sparse probe grid with 4x4 octahedral irradiance maps. Tiny cubemap renders (16x16/face) instead of ray tracing. 2-4 probes updated per frame via round-robin. Visibility-weighted trilinear interpolation at shading time.
  Step-by-Step Experiment Plan: Indoor/outdoor scenes with dynamic lighting. Compare against no GI, baked lightmaps, SSAO. Measure PSNR/SSIM vs path-traced reference, per-frame cost, power.

paper 2:
Bandwidth-Optimal Shadow Mapping for Mobile:
  Title: Tiled Shadow Mapping: Bandwidth-Efficient Shadows for Mobile Tile-Based GPUs
  Problem Statement: Shadow mapping is bandwidth-intensive on mobile GPUs.
  Motivation: Redesign shadow computation to work within tile memory hierarchy.
  Proposed Method: Per-tile shadow analysis, compact shadow masks in tile memory, atlas packing, temporal caching.
  Step-by-Step Experiment Plan: Outdoor scenes with multiple lights. Compare against CSM, VSM, screen-space shadows. Measure quality, bandwidth, power.

Meta-review:
Paper 1 tackles a fundamental problem in mobile rendering (the lack of GI) with a creative combination of sparse probes and temporal amortization. The approach of using tiny cubemap renders instead of ray tracing is particularly well-suited for mobile. The visual impact of adding GI to mobile scenes could be transformative. However, the temporal lag from round-robin updates could be noticeable with fast-changing lighting.

Paper 2 addresses shadow bandwidth, a well-known bottleneck in mobile rendering. The tile-local shadow evaluation approach is technically elegant and directly leverages mobile GPU architecture. The temporal caching of static shadows is a practical optimization. Both the problem and solution are well-scoped.

Paper 1 is more ambitious and potentially more impactful, while Paper 2 is more practical and lower-risk. For a top graphics venue, the ambition and novelty of Paper 1 would likely be valued more highly.

The accepted paper is paper 1.
