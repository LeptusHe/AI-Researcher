Title: MobileSR: Lightweight Temporal Super-Resolution for Tile-Based Mobile GPUs

1. Problem Statement: Mobile games render at reduced resolution to meet frame rate targets, producing blurry or aliased output. Desktop temporal upsampling methods (DLSS, FSR2) are too computationally expensive for mobile GPUs.

2. Motivation: By designing a temporal upsampling algorithm specifically for mobile tile-based architectures, using jittered rendering with a lightweight recurrent filter that accumulates sub-pixel detail across frames, we can achieve high-quality upsampling with minimal compute overhead and memory usage.

3. Proposed Method: MobileSR renders at lower internal resolution with per-frame Halton sequence sub-pixel jitter. A lightweight compute shader accumulates samples into a history buffer at output resolution using motion vectors for reprojection. The accumulation uses confidence-weighted blending that detects disoccluded regions via depth comparison and handles fast motion via motion magnitude thresholding. For pixels with low confidence (no valid history), an edge-directed spatial upsampler provides fallback by interpolating along detected edges rather than across them. The pipeline adds 2 compute passes (reprojection + accumulation) and approximately 1MB of additional memory for the history buffer.

4. Step-by-Step Experiment Plan:
    - Step 1: Add Halton sequence sub-pixel jitter (16 samples) to the projection matrix.
    - Step 2: Generate per-pixel motion vectors combining camera and per-object motion, stored as RG16F.
    - Step 3: Implement history reprojection compute shader with bilinear sampling and depth-based disocclusion detection.
    - Step 4: Implement confidence-weighted accumulation with color-difference, motion-reliability, and depth-disocclusion factors.
    - Step 5: Implement edge-directed spatial fallback for low-confidence pixels.
    - Step 6: Test 2x (540p->1080p) and 4x (270p->1080p) upsampling on dynamic game scenes.
    - Step 7: Compare against bilinear, bicubic, FSR 1.0, and TAA. Measure PSNR/SSIM vs native, temporal flicker metric, frame time overhead, power consumption.

5. Test Case Examples:
    - Baseline (Bilinear 2x): Fast action game on Snapdragon 8 Gen 2, render 540p bilinear to 1080p: 60 FPS, PSNR 28.3 dB, visible blur and shimmer.
    - Proposed (MobileSR 2x): Same scene, MobileSR: 58 FPS (1.4ms overhead), PSNR 34.1 dB, sharp edges and stable details.

6. Fallback Plan: If temporal accumulation introduces unacceptable ghosting in fast-motion scenarios, investigate more aggressive history rejection, hybrid spatial-temporal approaches based on per-pixel motion magnitude, or publish analysis of temporal upsampling challenges specific to mobile rendering.
