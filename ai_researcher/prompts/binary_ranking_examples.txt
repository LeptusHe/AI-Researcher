Adaptive Tile-Based Shading for Mobile GPUs:
  Title: Adaptive Tile-Based Variable Rate Shading for Energy-Efficient Mobile Rendering
  Problem Statement: Mobile GPUs use tile-based rendering architectures but apply uniform shading rates across all tiles, wasting compute on low-complexity regions.
  Motivation: By analyzing per-tile content complexity using depth discontinuities, normal variation, and motion vectors at minimal cost, we can dynamically assign shading rates that maximize visual quality per watt.
  Proposed Method: A lightweight tile complexity analysis compute pass scores each tile and assigns 4x4 coarse, 2x2 coarse, or full-rate shading. Temporal smoothing prevents flickering.
  Step-by-Step Experiment Plan: Test on Sponza and Bistro scenes on Snapdragon 8 Gen 2, Mali-G715, and Apple A17 devices. Measure FPS, power, PSNR/SSIM.

paper 2:
Simple Resolution Scaling for Mobile:
  Title: Dynamic Resolution Scaling with Fixed Thresholds for Mobile Games
  Problem Statement: Mobile games need to maintain frame rate targets while preserving visual quality.
  Motivation: Current DRS approaches use simple frame time feedback to adjust resolution, but fixed thresholds lead to oscillation.
  Proposed Method: Monitor frame time over a sliding window and scale resolution up/down when the average exceeds thresholds. Use hysteresis to prevent oscillation.
  Step-by-Step Experiment Plan: Test resolution scaling on 3 mobile games. Measure average FPS and user survey results.

Temporal Super-Resolution for Mobile Games:
  Title: MobileSR: Lightweight Temporal Super-Resolution for Tile-Based Mobile GPUs
  Problem Statement: Mobile games render at reduced resolution, producing blurry output. Desktop temporal upsampling (DLSS, FSR2) is too expensive for mobile.
  Motivation: Design temporal upsampling specifically for tile-based mobile architectures using jittered rendering with lightweight recurrent filtering.
  Proposed Method: Render at lower resolution with sub-pixel jitter, accumulate into history buffer using motion vector reprojection with confidence-weighted blending and edge-directed spatial fallback.
  Step-by-Step Experiment Plan: Test 2x and 4x upsampling on mobile devices. Compare against bilinear, FSR 1.0, TAA. Measure PSNR/SSIM, frame time, power consumption.

Neural Texture Compression for Mobile:
  Title: MobileNTC: Neural Texture Compression Optimized for Mobile GPU Architectures
  Problem Statement: Mobile devices have limited GPU memory and bandwidth, making texture storage a bottleneck.
  Motivation: Design neural texture representation optimized for mobile GPU tile-based memory hierarchy.
  Proposed Method: Encode textures as compact latent grid + tiny decoder network (<50K params). Decoder uses 1x1 convolutions efficient on mobile GPUs. Hardware-aware FP16/INT8 quantization.
  Step-by-Step Experiment Plan: Compare against ASTC at various quality levels. Measure compression ratio, decompression speed, PSNR/SSIM/LPIPS, GPU memory savings.

Bandwidth-Optimal Shadow Mapping for Mobile:
  Title: Tiled Shadow Mapping: Bandwidth-Efficient Shadows for Mobile Tile-Based GPUs
  Problem Statement: Shadow mapping is one of the most bandwidth-intensive operations in mobile rendering.
  Motivation: Redesign shadow mapping to work within tile memory hierarchy, minimizing off-chip bandwidth.
  Proposed Method: Per-tile shadow analysis identifies needed shadow cascades. Compact shadow mask stored in tile memory. Single atlas with tight packing. Temporal caching for static shadow regions.
  Step-by-Step Experiment Plan: Test outdoor scenes with directional + point lights. Compare against CSM, VSM, screen-space shadows. Measure quality, bandwidth, power.

paper 2:
Simple Shadow Quality Reduction:
  Title: Adaptive Shadow Map Resolution for Mobile Frame Rate Targets
  Problem Statement: Shadow maps consume significant bandwidth on mobile GPUs.
  Motivation: Reducing shadow map resolution saves bandwidth but reduces quality uniformly.
  Proposed Method: Dynamically adjust shadow map resolution based on camera distance to shadow-casting lights. Closer lights get higher resolution maps.
  Step-by-Step Experiment Plan: Test shadow quality at various resolutions. Measure frame time improvement.
